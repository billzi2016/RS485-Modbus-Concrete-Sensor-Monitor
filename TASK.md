## 2. 架构约束任务

### Task 2.1 确认 Django 结构模式

- 目标：
  - 明确 Django 采用标准项目结构
- 要求：
  - 必须有 `manage.py`
  - 必须有项目级 `config/`
  - 必须有业务级 `app/`
  - 不使用魔改目录结构
- 结果：
  - 后续开发统一按标准 Django 模式执行

### Task 2.2 确认采集器独立结构

- 目标：
  - 明确采集器不直接塞进 Django 请求流程
- 要求：
  - `collector/` 作为独立模块存在
  - 负责 asyncio 调度、Modbus TCP 通信、Redis 写入
- 结果：
  - 采集层与展示层职责明确

### Task 2.3 确认 Redis 作为中间缓存层

- 目标：
  - 明确采集器写 Redis，Django 读 Redis
- 要求：
  - 不在第一阶段引入关系型数据库
  - 最新快照和最近 10 分钟历史统一经 Redis 传递
- 结果：
  - 采集与展示彻底解耦

## 12. 当前结论

当前项目可以视为三层推进：

1. 先把文档和任务拆清楚
2. 再按最传统 Django 方式搭主程序骨架
3. 最后逐步接入 collector、Redis、页面和接口

这样推进的好处是：

- 不容易写乱
- 结构稳定
- 便于后续逐步实现和修改
